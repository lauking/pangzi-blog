最近看到很多文章关于GO环境的配置，那么go需要配置哪些环境变量呢？根据官方文档GOPATH,GOROOT,GOBIN,GOOS的go环境变量配置

说明

GOROOT go的安装目录
GOBIN go安装目录下的bin目录，里面有可执行的程序
GOOS 这个变量是在编译代码时使用的
GOPATH 开发的工作目录
（注：这个不是Go安装目录）

下载安装包

windows默认安装在C:\go
linux和UNIX之类系统可指定安装目录

windows下配置

在环境变量中新建

  • GOROOT=C:\go
  • GOBIN=C:\go\bin
  • GOPATH=E:\workcode\golang
    并在PATH中添加 %GOBIN%;%GOPATH%

linux和UNIX下配

编辑用户目录下的.bashrc文件(其他用户无权操作)

在文件末尾添加如下语句

  • export GOROOT=$home/go
  • export GOBIN=$GOROOT/bin
  • export GOPATH=$home/app/mygo
  • export PATH=$PATH:$GOBIN:$GOPATH

编辑完后执行：source .bashrc 是环境变量生效

编辑/etc/profile文件

在文件末尾添加如下语句

  • export GOROOT=$home/go
  • export GOBIN=$GOROOT/bin
  • export GOPATH=$home/app/mygo
  • export PATH=$PATH:$GOBIN:$GOPATH

编辑完后执行：source /etc/profile 是环境变量生效

创建工作空间

在设定的GOPATH路径下创建三个目录：

  • src 存放源代码（比如：.go .c .h .s等）
  • pkg 编译后生成的文件（比如：.a）
  • bin 编译后生成的可执行文件（为了方便，可以把此目录加入到 $PATH 变量中）

应用结构目录

建立包和目录：$GOPATH/src/mymath/sqrt.go（包名：”mymath”）
以后自己新建应用或者一个代码包都是在src目录下新建一个文件夹，文件夹名称一般是代码包名称，当然也允许多级目录，例如在src下面新建了目录$GOPATH/src/github.com/lauking/blog 那么这个包路径就是“/github.com/lauking/blog ”，包名称是最后一个
目录blog
执行如下代码 sh cd $GOPATH/src mkdir mymath 新建文件sqrt.go，内容如下 

go //
$GOPATH/src/mymath/sqrt.go源码如下： package mymath
func Sqrt(x float64) float64 {
    z := 0.0
    for i := 0; i < 1000; i++ {
        z -= (z*z - x) / (2 * x)
    }
    return z
}

这样我的应用包目录和代码已经新建完毕，注意：一般建议package的名称和目录名保持一致
